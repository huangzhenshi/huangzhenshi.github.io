设计模式

  • 单例模式：懒汉式、饿汉式、双重锁。通过private构造函数，提供统一的静态方法实例化全局唯一的静态变量。
  • 代理模式：简单封装、静态代理、动态代理、Cglib代理实现代理一个类、代理一种接口实现类的一个方法、代理一种接口实现类全部方法、所有类的所有非final方法，来提供类的额外的功能
  • 工厂模式：简单工厂模式、工厂方法模式、抽象工厂模式，实现解耦和隐藏架构构造过程。

排序算法

注意事项:

  1. 空间复杂度，选择排序、冒泡、插入排序是平方级别，希尔是1-2次方级别，其他的快排、归并、堆排序平均都是NlogN
  2. 算法要结合实际情况：比如数组已经基本有序，交换、比较、查找的计算机成本，内存是否充裕等

经典算法:

  • 选择排序：每次记录最大值的下标，交换一次。交换次数最少且稳定但比较次数太多，无法利用上一次比较的结果
  • 插入排序：维护一个递增的数组，适合基本有序的数组，基数不大的时候是最快的排序
  • 希尔排序：插入的加强版，先按照一定间隔进行排序，最后插排。空间复杂度从平方到小于平方。
  • 快排：首位做哨兵，找到在数组中的位置交换，递归。改进的算法，递归到小数组时还插入排序快。数组越随机越好
  • 归并排序：把数组两两切分，再把有序数组两两归并，浪费额外的内存。
  • 堆排序：初始化构建优先队列swim（），挨个从堆顶删除数据从末尾填充sink()到数组中。

IO流

  • 字节流处理数据基本单位是字节，字符流处理数据基本单位是2字节。
  • IO流要记住的类：inputStream、outputStream，Reader、Writer，File类
  • 细节:
  1. inputStream read()方法，实现类 FileInputStream 和 BufferedInputStream
  2. outputStream write()；实现类 FileOutputStream 和 BufferedOutputStream
  3. Reader抽象 读字符FileReader
  4. Writer抽象类 FileWriter
  5. File类用来对文件夹和文件进行增删改查操作isFile()、isDirectory()、exits()、file.list()

NIO区别：

  1. IO流是阻塞的，当前线程会等待读取或写结束再继续执行下面的代码，NIO的channel是可以选择是否阻塞的，一个线程可以管理多个channel
  2. IO流是面向流的，NIO是面向缓冲区的，操作缓冲区支持更加灵活的操作
  3. NIO适合聊天室,每个流数据量小但是连接数多，IO相反
  4. NIO中Channel是双向的，和Buffer交互，可以读可以取，IO中流是单向的

NIO 常见类

  • FileChannel，只能选择阻塞模式，读取和操作文件。特别是transfer方法，复制文件简洁
  • Buffer接口 ByteBuffer DoubleBuffer FloatBuffer
  • Selector 注册在channel上，可以操作多个channel

NIO.2

  • 支持异步IO，也就是回调式IO，性能提升，把流操作和回调函数告诉OS，然后就接着干别的事情了。
  • 支持更多灵活简洁的API操作,比如复制、移动、监听文件变动、递归目录：Path、Files类

Get | Post请求区别

  • 用途：get请求一般用作查询，post请求一般用作提交数据
  • 特点：get请求的参数在url里，暴露信息、长度限制、编码困难、会被缓存，post请求的参数在request body里面
  • 本质: get请求是产生一个TCP数据包，Post请求时2个TCP数据包，第一次传递header包括URL，第二个传递data

TCP/IP 协议

  • 三次握手：客户端向服务端发起请求，服务端确认收到请求确认是否能接收，客户端确认能接收并反馈给服务端。通信通道建立
  • 四次挥手：客户端向服务端发起断开请求，服务端确认收到请求并且把缓存的数据全部返回给客户端，服务端发送同意终止，客户端收到2次请求后确认断开并告知服务器
  • TCP |UDP 网络通讯用的：TCP用来可靠的数据交互，UDP只负责广播，不保证每个人都听的到

http请求和https请求

http请求的交互信息是明文，抓包的话，可以知道内容，Https给予SSL加密，交互的信息都是加密过的信息。需要申请SSL证书配置。

MongoDB

特点：

  • 为大数据而生，支持GB PB级大文本的存储和快速检索，灵活的扩展性、高可用、负载均衡
  • 丰富的索引：位置索引、全文索引、联合索引、普通索引
  • 不支持join和事务，其他mysql的功能都支持，执行计划、条件查询、分组和排序
  • schemless、value值可以是一个json对象或者集合
  • 文档型数据库
